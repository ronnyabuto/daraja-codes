1019 STK Push

Transaction Expired

The STK Push transaction exceeded the allowable processing time before the user responded.

Likely Causes
  • User did not respond to the STK prompt before it timed out (typically 1–3 minutes depending on phone model)
  • User's phone was slow to display the prompt due to network lag
Fix

Inform the user their payment request expired and offer them a button to initiate a fresh STK push. Implement a polling loop using the STK Push Query API (stkpushquery endpoint) to catch this state proactively before relying solely on the callback.

Notes

Different from error 1037 — 1037 means the prompt never reached the phone. 1019 means the prompt arrived but the user did not act on it in time.
