1032 STK Push

Request Cancelled by User

The user deliberately or accidentally dismissed the STK/USSD payment prompt.

Likely Causes
  • User pressed Cancel on the STK prompt
  • User ignored the prompt until it auto-dismissed
Fix

Re-initiate the STK push request. No action needed on the backend — this is a user-side cancellation.
