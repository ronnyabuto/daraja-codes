Daraja M-Pesa API — Error FAQ

Direct answers to the questions developers actually search for.

What does Daraja error 1037 mean?

Error 1037 (MSISDN Unreachable / No Response From User) means the STK Push prompt was sent but M-Pesa received no response from the user's handset within the timeout window. Common causes: phone is off, out of coverage, outdated SIM, or iOS eSIM issues. This is the most Googled Daraja error. Do not treat it as fatal — always offer a retry button in your UI. Why did my STK Push return error 1032?

Error 1032 (Request Cancelled by User) means the user dismissed the STK prompt — either pressed Cancel or let it auto-dismiss. This is a normal user action. Re-initiate the STK Push if the user wants to retry — no backend action needed. Full details →

What causes error 500.001.1001 (Wrong credentials / MerchantValidate failed)?

Two root causes share this code. (1) Wrong credentials: the Timestamp used to build the Password doesn't match the Timestamp in the request body — generate it once and reuse the same variable for both. (2) Merchant does not exist: the BusinessShortCode isn't registered on the platform. Read the full errorMessage field to know which you're dealing with. Full details →

How do I fix error 400.002.02 (Bad Request — Invalid Request Parameter)?

Read the errorMessage suffix — it names the exact invalid field. Invalid BusinessShortCode in Python: switch from data=payload to json=payload. Invalid Timestamp: format must be YYYYMMDDHHmmss with no separators. Invalid Amount: must be a positive integer, no decimals. Full details →

What is error 2001 in Daraja B2C?

Error 2001 (Invalid Initiator Information) means your operator credentials are wrong. Most commonly: SecurityCredential was encrypted using a stale G2 certificate from a GitHub repo. Download a fresh certificate directly from your Daraja portal, re-encrypt, and verify that InitiatorName exactly matches the portal. Full details →

Why is my Daraja access token invalid after going live?

All four credentials must be live simultaneously: Consumer Key, Consumer Secret, Passkey, and base URL (https://api.safaricom.co.ke/). If all four are correct and it still fails, Safaricom has likely not completed internal approval of your app. Email apisupport@safaricom.co.ke with your app ID and shortcode. This is not a code issue 90% of the time. Full details →

Why are my C2B sandbox callbacks not arriving?

C2B sandbox callbacks are unreliable by design — a known, long-standing Safaricom limitation with roughly 40% delivery rate. The developer community consensus: test C2B against a live deployment (Railway, VPS, etc.) with small real transactions. Localhost and ngrok URLs are blocked on production. Full details →

What does error 1019 (Transaction Expired) mean?

Error 1019 means the STK prompt reached the user's phone but they didn't act before the timeout (typically 1–3 minutes). Unlike 1037 where the prompt never reached the phone, 1019 means it arrived. Offer the user a fresh STK Push. Use the STK Push Query API to catch this proactively before relying on callbacks. Full details →

What is Daraja error 404.001.03?

Error 404.001.03 (Access Token Expired or Missing) means the Bearer token in your Authorization header has expired or is absent. Tokens expire after exactly 3600 seconds. Fix: cache the token and its expiry time, refresh before expiry — don't regenerate on every request. Ensure the header is Bearer <token> with a space. Full details →

What is Daraja error 401.003.01?

Error 401.003.01 (Invalid Access Token — OAuth Step) means the token was rejected at the authorization layer itself. This is distinct from 404.001.03 which fires during API calls. Most common cause: using a sandbox token against the production OAuth endpoint or vice versa. Regenerate with the correct credentials for the target environment. Full details →

What does error 9999 mean in Daraja?

Error 9999 is an alias for error 1025 — both mean the STK prompt couldn't be delivered due to a system issue or an oversized TransactionDesc (max 182 characters). Check TransactionDesc length, then retry after 1–2 minutes. Handle 9999 and 1025 identically in your code. Full details →

What does "Invalid Initiator Info" mean in an STK Push callback?

"Invalid Initiator Info" in an STK Push callback means the user entered the wrong M-Pesa PIN. M-Pesa allows a maximum of 3 failed PIN attempts. Prompt the user to retry. Don't confuse this with error 2001 — that is a developer credentials issue in B2C, not a user PIN issue. Full details →

What is Daraja error 1 (Insufficient Funds)?

Error 1 means the user's M-Pesa balance is too low and their Fuliza overdraft limit is also exhausted or inactive. Inform the user they need to top up and retry. Full details →

What is Daraja error 1001 (Transaction In Progress)?

Error 1001 means the user already has an active USSD session — another M-Pesa transaction is in progress. M-Pesa can't lock the subscriber for a new transaction. Wait 1–2 minutes and retry. Full details →

What is Daraja error 1025?

Error 1025 means the STK prompt couldn't be sent — either a temporary M-Pesa system issue or TransactionDesc over 182 characters. Check the length first, then retry after 1–2 minutes. Error 9999 is the same — handle both identically. Full details →
